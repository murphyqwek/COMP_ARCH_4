## Язык программирования: Lisp-like

Язык представляет собой диалект Lisp с поддержкой глобальных переменных, функций, циклов и обработки прерываний.

### Форма Бэкуса-Наура (БНФ)
```
<program> ::= <expression>*
<expression> ::= <literal> | <symbol> | <list>
<list> ::= "(" <operator> <arg>* ")"
<operator> ::= "defvar" | "defun" | "setq" | "if" | "loop" | "print" | "call" | "+" | "-" | "*" | "mod" | "progn" | "def-interrupt"
<literal> ::= <integer> | <string>
<symbol> ::= [a-zA-Z_][a-zA-Z0-9_-]*
```

**Особенности семантики:**
- `defvar`: объявление глобальной переменной.
- `defun`: определение функции с локальными аргументами.
- `setq`: присваивание значения переменной.
- `loop`: цикл вида `(loop <var> <start> <end> <body>)`.
- `def-interrupt`: специальная форма для определения обработчика прерывания (вектор 0x0010).
- **Типы данных:** 32-битные знаковые целые числа и строки (в формате Pascal string: длина + символы).

## Организация памяти и регистры

- **Архитектура**: Фон-Неймана;
- **Машинное слово:** 32 бита, знаковое число;
- **Регистры:**
    - `R0, R1, R2, R3`: Регистры общего назначения.
    - `SP`: Указатель стека (используется для `PUSH`, `POP`, `CALL`, `RET` и прерываний).
- **Стек:** Растет вниз, начальное значение `SP = 9000`.

### Память
- Общий объем: 10000 слов.
- **0x0000 - 0x0006:** Зарезервировано: **0x00-0x01 - jmp на секцию кода, 0x02-0x03 - jmp на обработчик прерывания, 0x04-0x06 - MMIO порты**.
- **0x007+:** Секция данных.
- **После секции данных идет обработчик прерывания и секция кода**
- **MMIO порты:**
    - `0x04`: Ввод символа.
    - `0x05`: Вывод целого числа (Int Out).
    - `0x06`: Вывод символа (Char Out).

## Система команд (ISA)

Инструкции сериализуются в формат: `[Заголовок] [Аргумент 1] [Аргумент 2] ...`
- **Заголовок (32 бита):**
    - 0-7 биты: Opcode;
    - 8-11 биты: Количество аргументов (0-4);
    - 12-27 биты: Режимы адресации (по 4 бита на аргумент);

### Режимы адресации
1. `IMM` (Immediate): Значение операнда находится в следующем слове;
2. `MEM` (Absolute): Прямой адрес в памяти;
3. `REG` (Register): Работа с регистрами R0-R3, SP;
4. `REG_INDIRECT`: Значение в памяти по адресу, хранящемуся в регистре;

### Набор инструкций

|Мнемоника	|Опкод  |	Описание                                                                        |
|-----------|-------|-----------------------------------------------------------------------------------|
|NOP	    |0x00	|    Нет операции (пропуск такта)|
|HALT	    |0x01	|    Остановка работы процессора|
|IRET	    |0x02	|    Возврат из обработчика прерывания (восстановление PC, флагов и регистров)|
|RET	    |0x03	|    Возврат из подпрограммы (извлечение PC из стека)|
|MOV	    |0x10	|    Копирование данных между регистрами, памятью или константой|
|PUSH	    |0x11	|    Поместить значение в стек|
|POP	    |0x12	|    Извлечь значение из стека|
|ADD	    |0x20	|    Сложение операндов|
|ADC	    |0x21	|    Сложение с учетом флага переноса|
|SUB	    |0x22	|    Вычитание операндов|
|SBC	    |0x23	|    Вычитание с учетом флага переноса|
|MUL	    |0x24	|    Перемножение операндов|
|MOD	    |0x25	|    Вычисление остатка от деления|
|CMP	    |0x26	|    Сравнение операндов |
|JMP	    |0x30	|    Безусловный переход по адресу|
|JEQ	    |0x31	|    Переход по адресу, если установлен флаг нуля (Z=1)|

|CALL	    |0x33	|    Вызов подпрограммы (сохранение текущего PC в стек и переход)|

## Микроархитектура и Суперскалярность

Модель процессора имитирует **суперскалярное выполнение**:

1. **Fetch (Выборка):** На каждом шаге Control Unit считывает команду за 1 такт. Потом происходит считывание аргументов (1 аргумент за 1 такт). Затем считывает вторую команду (если первая команда позволяет выполнить две команды одноврееменно).
2. **Анализ зависимостей ():**
   - Проверяются конфликты по данным (RAW, WAW, WAR).
   - Проверяются конфликты по флагам АЛУ.
   - Проверяются конфликты доступа к памяти.
3. **Параллельное исполнение:**
   - Если инструкции независимы, они выполняются за один цикл.
   - Если обнаружена зависимость, выполняется только одна инструкция (но вторая инструкция остаеется в буффере).

## Прерывания (Trap)

Система поддерживает аппаратные прерывания по расписанию:
1. При наступлении нужного тика и установленном флаге `IE` (Interrupt Enable), CU сохраняет `PC`, флаги и регистры `R0-R3` в стек.
2. Флаг `IE` сбрасывается.
3. Символ внешнего устройства помещается в ячейку `0x04`.
4. Управление передается по адресу `0x02`.
5. Завершается обработка инструкцией `IRET`.

## Транслятор

- Проходит по lisp коду и ищет переменные и строки;
- Аллоцирует переменные и строки в секции данных (с адреса `0x07`);
- Проходится по коду и генерирует машинный код;
- Вывод результата в бинарный файл (.bin) и дизассемблировнный лог (.bin.log).

**Пример запуска:**
```bash
python translator_lisp.py examples/prob1.lisp examples/prob1.bin
```

### Тестирование через Golden Tests:
```bash
pytest .\golden_test.py -v  
```
Тесты проверяют:
1. `cat`: ввод-вывод через прерывания;
2. `hello`: вывод 'Hello World!';
3. `hello_user_name`: запрос имени и его вывод;
4. `double`: арифметика двойной точности;
5. `fact`: вычисление факториала 6 (пример рекурсии);
6. `sort`: алгоритм сортировки в памяти;
7. `test`: скрипт-пример, что любое statement - expression;
8. `prob2`: решение `Euler problem 6`;
