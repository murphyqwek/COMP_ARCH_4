## Язык программирования: Lisp-like

Язык представляет собой диалект Lisp с поддержкой глобальных переменных, функций, циклов и обработки прерываний.

### Форма Бэкуса-Наура (БНФ)
```
<program>     ::= <expression>*
<expression>  ::= <literal> | <symbol> | <list>
<list>        ::= "(" (<operator> | <symbol>) <expression>* ")"

<operator>    ::= "defvar" | "defun" | "setq" | "if" | "loop"
                | "print" | "print-pstr" | "print-char" | "in-char"
                | <arith-op> | <cmp-op>
                | "read-mem" | "write-mem"
                | "progn" | "def-interrupt"

<arith-op>    ::= "+" | "-" | "*" | "/" | "mod" | "adc" | "sbc"
<cmp-op>      ::= "=" | ">" | "<"

<literal>     ::= <integer> | <string>
<symbol>      ::= [a-zA-Z_][a-zA-Z0-9_-]*
<integer>     ::= [-+]?[0-9]+
<string>      ::= '"' [^"]* '"'
```

**Особенности семантики:**
- `defvar`: объявление глобальной переменной.
- `defun`: определение функции с локальными аргументами.
- `setq`: присваивание значения переменной.
- `loop`: цикл вида `(loop <var> <start> <end> <body>)`.
- `def-interrupt`: специальная форма для определения обработчика прерывания.
- **`+`**: бинарный по умолчанию (`(+ a b)`), но если переданы **3+ целочисленных литерала** (`(+ 1 2 3 4 5)`), транслятор сворачивает выражение в одну инструкцию `NADD`. Прочие арифметические операторы остаются строго бинарными.
- **Типы данных:** 32-битные знаковые целые числа и строки (в формате Pascal string: длина + символы).

## Организация памяти и регистры

- **Архитектура**: Фон-Неймана;
- **Машинное слово:** 32 бита, знаковое число;
- **Регистры:**
    - `R0, R1, R2, R3`: Регистры общего назначения.
    - `SP`: Указатель стека (используется для `PUSH`, `POP`, `CALL`, `RET` и прерываний).
    - `TEMP`: Внутренний буфер CU, используемый микропрограммами для хранения промежуточного значения ALU между шагами (например, `MOV mem→mem`, арифметика с двумя операндами в памяти, NADD при записи в память). **Программно недоступен**, не входит в register file, видимый ассемблеру.
- **Стек:** Растет вниз, начальное значение `SP = MEMORY_SIZE − 1`.
- **Инструкция:**
    - `IR`: Хранит машинное слово инструкции
    - `OP1, OP2, OP3`: Хранят операнды для инструкции. Для инструкций с >3 операндов (`NADD`) `OP3` дополнительно используется как поток: каждый источник, начиная с третьего, последовательно подгружается в `OP3` уже на стадии execute.

### Память
- Общий объем: 1024 слова (`MEMORY_SIZE`).
- **0x0000 – 0x0006:** Зарезервировано: **0x00-0x01 – jmp на секцию кода, 0x02-0x03 – jmp на обработчик прерывания, 0x04-0x06 – MMIO порты**.
- **0x0007+:** Секция данных.
- **После секции данных идёт обработчик прерывания и секция кода.**
- **MMIO порты:**
    - `0x04`: Ввод символа с прерывания (`TRAP_BUFFER`).
    - `0x05`: Вывод целого числа (`OUT_INT`).
    - `0x06`: Вывод символа (`OUT_CHAR`).

## Система команд (ISA)


| 31.....28 | 27.....24 | 23.....20 | 19.....16 | 15.....12 | 11......8 | 7.......0 |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| arg4_mode | arg3_mode | arg2_mode | arg1_mode | arg_count | opcode | opcode (low) |

- **Заголовок (32 бита):**
    - 0–7 биты: Opcode;
    - 8–11 биты: Количество аргументов (0–15);
    - 12–27 биты: Режимы адресации (по 4 бита на аргумент, до 4 слотов);
    - 28–31 биты: Зарезервировано.
- **Замечание про NADD:** инструкция принимает до 15 операндов (1 dst + до 14 источников). В заголовке умещаются режимы только первых 4 операндов; по соглашению ISA все источники `NADD` всегда имеют режим `IMM`, поэтому декодер игнорирует биты режимов для аргументов 5+ и считает их `IMM`.

### Режимы адресации
1. `IMM` (Immediate): Значение операнда находится в следующем слове;
2. `MEM` (Absolute): Прямой адрес в памяти;
3. `REG` (Register): Работа с регистрами `R0-R3`, `SP`;
4. `REG_IND`: Значение в памяти по адресу, хранящемуся в регистре.

### Подробное объяснение тактов команд
Для арифметических команд (ADD/SUB/MUL/MOD/ADC/SBC/DIV) и MOV/CMP количество тактов зависит от сочетания режимов адресации **Назначения (Dst)** и **Источника (Src)**:

| Режим Dst \ Режим Src | REG | IMM | MEM | REG_IND |
| :--- | :---: | :---: | :---: | :---: |
| **REG** | 1 | 1 | 2 | 3 |
| **MEM** | 2 | 2 | 4 | 5 |
| **REG_IND** | 3 | 3 | 5 | 6 |

#### Логика для арифметики (ADD, SUB, MUL, ADC, SBC, DIV, MOD)
Такты вычисляются как сумма стоимости каждого операнда:
1. **Источники (Args):** `REG/IMM` = +1 такт, `MEM` = +2 такта, `REG_IND` = +3 такта.
2. **Назначение (Dst):** `REG` = +0 тактов, `MEM` = +2 такта, `REG_IND` = +3 такта.

*Пример: `ADD [R0], R1, [100]` (REG_IND, REG, MEM) = 3 (Dst) + 1 (Src1) + 2 (Src2) = **6 тактов**.*

#### Логика для NADD
Стоимость складывается из:
1. Подготовки приёмника:
   - `Dst = REG` → +1 такт (запись первого сложения сразу в регистр);
   - `Dst = MEM` → +2 такта (`stage addr` + первая запись);
   - `Dst = REG_IND` → +2 такта.
2. На каждый источник, начиная с третьего, добавляется:
   - `Dst = REG` → +2 такта (`fetch` следующего src в `OP3` + `add`);
   - `Dst = MEM/REG_IND` → +3 такта (`fetch` + `TEMP ← [Dst]` + `[Dst] ← TEMP + OP3`).

*Пример: `NADD [7], #1, #2, #3, #4, #5` (Dst=MEM, N=5) = 2 (`stage` + первая запись `1+2`) + 3·3 (по три такта на каждый из src3, src4, src5) = **11 тактов исполнения** (без учёта fetch IR + первых 3 операндов).*

#### Логика вызова INT
Данная команда загружается в IR на этапе FETCH, если есть сигнал IRQ.

## Прерывания (Trap)

Система поддерживает аппаратные прерывания по расписанию:
1. При наступлении нужного тика и установленном флаге `IE` (Interrupt Enable), CU сохраняет `PC`, флаги и регистры `R0-R3` в стек.
2. Флаг `IE` сбрасывается.
3. Символ внешнего устройства помещается в ячейку `0x04`.
4. Управление передаётся по адресу `0x02`, где содержится `JMP` на функцию прерывания.
5. Завершается обработка инструкцией `IRET`.

## Транслятор

- Проходит по lisp коду и ищет переменные и строки;
- Аллоцирует переменные и строки в секции данных (с адреса `0x07`);
- Проходится по коду и генерирует машинный код;
- При компиляции выражения `(+ ...)` транслятор проверяет аргументы: если их 3+ и **все** являются целыми литералами, выражение сворачивается в одну инструкцию `NADD`; иначе используется обычный бинарный `ADD` (для `(+ a b)` или вложенных выражений с переменными).
- Вывод результата в бинарный файл (.bin) и дизассемблированный лог (.bin.log).

**Пример запуска:**
```bash
python translator_lisp.py examples/prob1.lisp examples/prob1.bin
```
